## Requisitos Não Funcionais (RNF)

### RNF01 — Login
O Login deve ser realizado rapidamente dentro de 2 segundos.

### RNF02 — Agendamento
O tempo maximo para a confirmação de um agendamento deve ser de 3 segundos.

### RNF03 — Disponibilidade
O sistema deve ficar disponível 24h por dia 7 dias por semana.

### RNF04 — Criptografia
A senha dos usuários deve ser salva no banco de dados de forma criptografada.

### RNF05 - Agendamento
O usuário deve conseguir realizar o agendamente em no maximo 4 passos.

### RNF06 - Compatibilidade
O sistema deve ser compativel com S.O Android e IOS.

### RNF07 - Escalabilidade
O sistema deve suportar simultaneamente pelo meno 25 usuários sem perda de perfomance. 

### RNF08 - Manutenabilidade
O código do sistema deve seguir padrões de desenvolvimento e documentação, facilitando manutenção futura.
